# 关键需求

## 一、账户与身份域

### 用户注册与登录

- 邮箱/手机号唯一性校验
- 登录方式：学号+密码、手机号+密码、邮箱+密码
- 密码强度校验与重置（邮件/短信找回）

### 用户资料与地址管理

- 资料查看/修改：昵称、联系方式、邮箱等
- 地址簿：增删改查、默认地址设置

### 角色与权限

- 角色：普通用户、跑腿员、管理员
- 资源级与操作级权限控制（RBAC），审计日志

## 二、食堂与菜品域

### 菜品管理

- 条件过滤：食堂、价格区间、标签、评分
- 排序：综合、评分
- 菜品管理：名称、价格、图片等
- 评分模型与可追溯性（防刷评价）

### 智能推荐
  
- 通过评分等综合排序进行推荐

## 三、订单与交易域

- 跑腿订单撮合
  - 用户发单：取货地、送达地、报酬、时效、备注、安全提示
  - 跑腿员接单：可接单列表、冲突校验（并发接单）
  - 订单确认收货与评价（双向）
- 账户余额与分账
  - 充值与提现：余额充值（校园卡/第三方），跑腿员余额提现（到账周期、限额）
  - 分账结算：平台抽佣、跑腿员收益、食堂商户结算周期

## 四、跑腿员域

- 认证与审核
  - 资料提交：身份证号、学生证、健康证、手机号实名
  - 管理员审核工作流与复审
- 跑腿员工作台
  - 可接订单地图视图、抢单策略（距离/信誉/历史完成率）
  - 我的订单：进行中/已完成/异常
  - 收益与信誉：信誉分计算、申诉入口、违规扣分规则

## 五、评价与信誉体系

- 双向评价
  - 用户对跑腿员、跑腿员对用户，星级+标签+文本
  - 评价时间窗与追评机制
- 信誉分与权重
  - 维度：准时率、取消率、投诉率、完成量、评分
  - 影响：排序优先级、接单限制、奖励/惩罚

## 六、平台管理后台（Admin）

- 用户/跑腿员/菜品管理
- 审核流与风控处置
- 内容审核（评价、图片）

## 七、非功能关键项

- 架构与性能
  - 前后端分离+微服务：用户、菜品、订单、跑腿、推荐
  - 水平扩展，目标并发≥1000在线，P95响应≤2秒
  - 消息队列：异步通知、订单状态事件、推荐行为日志
- 数据一致性与事务
  - 订单状态迁移与余额变动强一致
- 合规与内容治理
  - 隐私政策与用户授权
  - 评价内容审核规范与申诉通道

## 八、接口与前端关键页面

- 页面
  - 首页推荐、菜品列表与详情、订单列表/详情、跑腿发单、跑腿接单、消息中心、个人中心、地址簿、钱包/充值/提现
- 管理端页面
  - 审核中心（跑腿员/商户）、订单、用户与权限、内容审核

## 九、里程碑与优先级

- M1（MVP）：注册登录、食堂菜品浏览、下单支付、订单状态、地址簿、基础通知、基础评价、管理员最小闭环
- M2：跑腿认证与接单、分账结算、余额充值提现、信誉体系V1
- M3：个性化推荐、风控与风评优化、数据看板
