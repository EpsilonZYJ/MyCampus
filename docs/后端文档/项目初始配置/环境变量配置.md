# 🔐 环境变量配置说明

## 概述

为了提高安全性，本项目使用 `.env` 文件来管理敏感配置信息，特别是数据库密码。

## 配置步骤

### 1. 创建环境变量文件

```bash
# 复制模板文件
cp .env.example .env
```

### 2. 编辑环境变量

```bash
# 使用你喜欢的编辑器编辑 .env 文件
nano .env
# 或者
vim .env
```

### 3. 修改关键配置

**重要：请务必修改以下默认值：**

```bash
# 修改 MongoDB root 密码
MONGO_INITDB_ROOT_PASSWORD=your_secure_password_here

# 修改应用数据库用户密码
MYCAMPUS_DB_PASSWORD=your_app_secure_password_here
```

## 环境变量说明

| 变量名 | 说明 | 默认值 | 是否必需 |
|--------|------|--------|----------|
| `MONGO_INITDB_ROOT_USERNAME` | MongoDB 管理员用户名 | `admin` | ✅ |
| `MONGO_INITDB_ROOT_PASSWORD` | MongoDB 管理员密码 | `password123` | ✅ |
| `MONGO_INITDB_DATABASE` | 初始化数据库名 | `mycampus` | ✅ |
| `MYCAMPUS_DB_USER` | 应用数据库用户名 | `mycampus_user` | ✅ |
| `MYCAMPUS_DB_PASSWORD` | 应用数据库密码 | `mycampus_password` | ✅ |
| `SPRING_PROFILES_ACTIVE` | Spring Boot 配置文件 | `docker` | ❌ |
| `SERVER_PORT` | 应用内部端口 | `8080` | ❌ |
| `MONGODB_PORT` | MongoDB 映射端口 | `27017` | ❌ |
| `APP_PORT` | 应用映射端口 | `8080` | ❌ |

## 安全建议

### 🔒 密码安全
- 使用长度至少 12 位的复杂密码
- 包含大小写字母、数字和特殊字符
- 避免使用常见词汇或个人信息

### 📁 文件保护
- `.env` 文件已被添加到 `.gitignore`，不会被提交到版本控制
- 确保服务器上的 `.env` 文件权限设置为 `600` (仅所有者可读写)

```bash
chmod 600 .env
```

### 🚫 注意事项
- **永远不要**将 `.env` 文件提交到 Git 仓库
- **永远不要**在代码中硬编码密码
- **定期更换**生产环境密码

## 验证配置

运行环境检查脚本来验证配置：

```bash
./scripts/backend-check-env.sh
```

## 部署

配置完成后，使用部署脚本：

```bash
./scripts/backend-deploy.sh
```

或直接使用 Docker Compose：

```bash
docker-compose up -d
```

## 生产环境额外配置

### 环境变量优先级

1. 系统环境变量 (最高优先级)
2. `.env` 文件
3. `docker-compose.yml` 中的默认值 (最低优先级)

### 推荐的生产环境配置

```bash
# 生产环境 .env 示例
MONGO_INITDB_ROOT_USERNAME=prod_admin
MONGO_INITDB_ROOT_PASSWORD=very_secure_password_with_special_chars!@#
MONGO_INITDB_DATABASE=mycampus_prod
MYCAMPUS_DB_USER=mycampus_prod_user
MYCAMPUS_DB_PASSWORD=another_very_secure_password_123!
SPRING_PROFILES_ACTIVE=production
```

## 故障排查

### 常见问题

1. **权限错误**
   ```bash
   chmod +x backend-deploy.sh backend-check-env.sh
   ```

2. **端口冲突**
   - 检查端口是否被占用：`lsof -i :8080`
   - 修改 `.env` 文件中的端口配置

3. **MongoDB 连接失败**
   - 确保密码中没有特殊字符需要 URL 编码
   - 检查 MongoDB 容器是否正常启动

### 获取帮助

如果遇到问题，请检查：
- Docker 和 Docker Compose 日志
- `.env` 文件格式是否正确
- 网络连接是否正常
