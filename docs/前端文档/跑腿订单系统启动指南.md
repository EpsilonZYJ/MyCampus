# 跑腿订单系统 - 启动和测试指南

## 项目概述

本项目为校园跑腿订单系统，采用前后端分离架构，包含以下核心功能：

### 学生功能
- 创建跑腿订单（填写取件地点、送达地点、联系电话、跑腿费用等）
- 查看个人订单列表（活跃订单/已完成订单）
- 删除已完成的订单
- 申请成为跑腿员（提交身份证号和学生证照片）

### 跑腿员功能
- 查看待接订单列表（所有待接单的订单）
- 接单并更新订单状态（已接单 → 配送中 → 已完成）
- 查看个人接单记录

### 管理员功能
- 审核跑腿员申请（批准/拒绝）
- 查看待审核申请列表
- 管理跑腿员权限

## 技术栈

### 后端
- **Spring Boot** 3.5.6
- **MongoDB** 数据库（端口：27017）
- **Maven** 构建工具
- **Java** 17

### 前端
- **React** 19.x
- **React Router DOM** 7.9.4（路由管理）
- **Vite** 构建工具
- **Font Awesome** 4.7.0（图标库）

### 特色功能
- **角色切换系统**：支持在学生、跑腿员、管理员角色间切换（使用 UserContext + LocalStorage 持久化）
- **权限控制**：基于当前激活角色的动态菜单显示
- **响应式设计**：渐变主题 UI（学生-紫色、跑腿员-粉色、管理员-橙色、申请-蓝色）
- **Tab 切换**：订单状态筛选（活跃订单/已完成订单）

3. 直接启动项目即可测试

### 方式二：完整部署（从零开始）

如果需要重新初始化数据库，请按以下步骤操作。

### 1. 启动 MongoDB 数据库

确保 MongoDB 已安装并正在运行：

```bash
# macOS (使用 Homebrew)
brew services start mongodb-community

# 或手动启动
mongod --config /usr/local/etc/mongod.conf

# 验证 MongoDB 是否运行
mongo --eval "db.version()"
```

如果使用 Docker：
```bash
docker-compose up -d
```

### 2. 启动后端服务

在项目根目录下：

```bash
cd backend/MyCampus

# 方式一：使用 Maven Wrapper（推荐）
./mvnw clean install
./mvnw spring-boot:run

# 方式二：如果已安装 Maven
mvn clean install
mvn spring-boot:run
```

后端服务将在 `http://localhost:8080` 启动

**验证后端是否启动成功：**
```bash
# 测试健康检查（如果有的话）
curl http://localhost:8080/api/dishes

# 应该返回空数组或现有数据
```

### 3. 启动前端服务

在新的终端窗口中：

```bash
cd frontend

# 安装依赖（首次运行时需要）
npm install

# 启动开发服务器
npm run dev
```

前端服务将在 `http://localhost:5173` 启动（Vite 默认端口）

浏览器会自动打开，如果没有，请手动访问：`http://localhost:5173`

### 4. 使用角色切换功能

前端右上角有角色切换菜单：
- 点击用户名旁的下拉箭头
- 选择要切换的角色（学生/跑腿员/管理员）
- 导航菜单会根据当前角色动态更新

## 🛠️ 常见问题与解决方案

### 1. 后端启动失败 - 无法连接 MongoDB

**错误信息：** 
```
Failed to connect to MongoDB at localhost:27017
com.mongodb.MongoSocketOpenException
```

**解决方案：**
```bash
# 检查 MongoDB 是否运行
brew services list | grep mongodb

# 如果未运行，启动 MongoDB
brew services start mongodb-community

# 或手动启动
mongod --config /usr/local/etc/mongod.conf

# 验证连接
mongosh --eval "db.version()"
```

**配置检查：**
- 文件路径：`backend/MyCampus/src/main/resources/application.properties`
- 确认配置：`spring.data.mongodb.uri=mongodb://localhost:27017/mycampus_db`

---

### 2. 前端 API 请求失败 - CORS 错误

**错误信息：** 
```
Access to fetch at 'http://localhost:8080/api/errand-orders' 
from origin 'http://localhost:5173' has been blocked by CORS policy
```

**解决方案：**

后端已配置 CORS，检查 `CorsConfig.java` 是否存在：

```java
// backend/MyCampus/src/main/java/com/mycampusdev/mycampus/config/CorsConfig.java
@Configuration
public class CorsConfig {
    @Bean
    public CorsFilter corsFilter() {
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        CorsConfiguration config = new CorsConfiguration();
        config.setAllowCredentials(true);
        config.addAllowedOrigin("http://localhost:5173");
        config.addAllowedHeader("*");
        config.addAllowedMethod("*");
        source.registerCorsConfiguration("/**", config);
        return new CorsFilter(source);
    }
}
```

如果问题仍存在，尝试：
1. 清除浏览器缓存
2. 重启后端服务
3. 检查浏览器控制台的实际请求 URL

---

### 3. 订单创建失败 - Customer not found

**错误信息：** 
```json
{
  "code": 404,
  "message": "Customer not found with id: xxx",
  "data": null
}
```

**原因分析：**
- 前端使用的 userId 在数据库中不存在
- MongoDB 字段映射问题（userName vs user_name）

**解决方案：**

检查前端配置的用户 ID：
```javascript
// frontend/src/constants/roles.js
export const MOCK_USERS = {
  ROLE_STUDENT: {
    id: '68f4a41f21766212095f1b0a',  // 确认这个 ID 存在
    // ...
  }
}
```

在 MongoDB 中验证：
```javascript
mongosh mycampus_db
db.tb_user.findOne({_id: ObjectId("68f4a41f21766212095f1b0a")})
```

如果用户不存在，参考后面的"创建测试数据"章节。

---

### 4. 跑腿员无法接单 - Runner is not verified

**错误信息：** 
```json
{
  "code": 400,
  "message": "Runner is not verified",
  "data": null
}
```

**原因：** 跑腿员未通过管理员审核

**解决方案：**

方法一：通过管理员审核
1. 切换到管理员角色
2. 进入"跑腿员审核"页面
3. 批准该跑腿员申请

方法二：直接修改数据库
```javascript
mongosh mycampus_db
db.tb_user.updateOne(
  {_id: ObjectId("68f4a42421766212095f1b0b")},
  {$set: {"runnerProfile.isVerified": true}}
)
```

---

### 5. 角色切换后菜单不更新

**现象：** 切换角色后，导航栏菜单没有变化

**解决方案：**
1. 检查浏览器 LocalStorage：
   - 打开浏览器开发者工具 → Application → Local Storage
   - 查找 `currentUser` 和 `currentRole` 键
2. 清除缓存并刷新：
   ```javascript
   localStorage.clear()
   location.reload()
   ```
3. 确认 `UserContext` 正常工作：
   - 检查 `frontend/src/contexts/UserContext.jsx`
   - 确认 `currentRole` 状态正确更新

---

### 6. 订单状态显示异常

**现象：** 订单状态显示为英文（PENDING, ACCEPTED）而非中文

**原因：** 状态映射函数未正确调用

**解决方案：**

检查页面代码中的状态映射：
```javascript
const getStatusText = (status) => {
  const statusMap = {
    PENDING: '待接单',
    ACCEPTED: '已接单',
    IN_PROGRESS: '配送中',
    COMPLETED: '已完成',
    CANCELLED: '已取消'
  };
  return statusMap[status] || status;
};
```

---

### 7. Font Awesome 图标不显示

**现象：** 页面上的图标显示为方框或空白

**解决方案：**

检查 `index.html` 中是否引入了 Font Awesome：
```html
<!-- frontend/index.html -->
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/4.7.0/css/font-awesome.min.css">
```

或使用 NPM 安装：
```bash
npm install font-awesome
```

---

### 8. 响应格式不一致错误

**错误信息：** `订单创建失败: success`

**原因：** 前端检查 `result.success` 但后端返回格式为 `{code, message, data}`

**解决方案：**

已修复，确认所有 API 调用使用：
```javascript
// ✅ 正确
if (result.code === 200) {
  // 成功逻辑
}

// ❌ 错误（旧代码）
if (result.success) {
  // ...
}
```

```javascript
  roles: ["ROLE_STUDENT"],         // 角色数组
  
  // 跑腿员专属字段（仅当用户有 ROLE_RUNNER 时存在）
  runnerProfile: {
    status: "OFFLINE",             // 在线状态：ONLINE/OFFLINE/DELIVERING
    rating: 5.0,                   // 评分（1-5）
    totalOrdersCompleted: 0,       // 完成订单总数
    idCardNumber: "420101199001011234",      // 身份证号
    studentIDCardUrl: "http://example.com/card.jpg", // 学生证照片（新增）
    isVerified: true,              // 是否通过审核
    runnerSince: ISODate("2025-01-01T00:00:00Z"), // 成为跑腿员时间
    totalEarnings: NumberDecimal("0.00")          // 总收入
  },
  
  addresses: [],                   // 收货地址数组
  createdAt: ISODate("2025-01-01T00:00:00Z"),
  updatedAt: ISODate("2025-01-01T00:00:00Z")
}
```

**字段说明：**
- `userName`：MongoDB 存储为 camelCase（不是 user_name）
- `studentId`：建议创建唯一索引
- `roles`：可包含多个角色，如 `["ROLE_STUDENT", "ROLE_RUNNER"]`
- `runnerProfile.isVerified`：false 表示待审核，true 表示已批准
