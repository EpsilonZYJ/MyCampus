### 2. 功能特性

✨ **自动图片处理**
- 从 `imgs/` 目录读取图片文件
- 自动转换为 Base64 编码（Data URL 格式）
- 自动识别图片类型（jpg/png/gif/webp）
- 显示图片加载状态和文件大小

🛡️ **容错机制**
- 图片文件不存在时使用占位图片
- 显示友好的警告信息
- 不影响初始化流程继续执行

🔄 **向后兼容**
- 仍支持旧格式的 `image_data` 字段
- 可以混用 `image_path` 和 `image_data`

## 📝 使用说明

### 数据文件格式

**新格式** (推荐):
```json
{
    "dish_name": "宫保鸡丁",
    "restaurant": "百景园食堂",
    "image_path": "imgs/gongbao_chicken.jpg",
    "price": 28.8,
    ...
}
```

**旧格式** (仍支持):
```json
{
    "dish_name": "宫保鸡丁",
    "image_data": "data:image/jpeg;base64,/9j/4AAQ...",
    "image_type": "jpeg",
    ...
}
```

### 工作流程

```
1. 准备图片
   │
   ├── 方式 A: 添加真实图片到 imgs/
   ├── 方式 B: 运行 generate-placeholder-images.sh 生成占位图
   └── 方式 C: 跳过图片（使用默认占位图）
   │
2. 配置 init-data.json
   │
   └── 使用 image_path 引用图片文件
   │
3. 运行初始化
   │
   ├── npm run init           # 普通初始化
   └── npm run init:clear     # 清空重建
   │
4. 验证结果
   │
   └── 检查日志和数据库
```

### 执行示例

**有图片文件时**:

## 🎯 技术实现

### 图片处理流程

```javascript
// 1. 读取图片文件
const imagePath = path.resolve(dataDir, '..', dishData.image_path);
const imageBuffer = await fs.readFile(imagePath);

// 2. 转换为 Base64
const base64Data = imageBuffer.toString('base64');

// 3. 构建 Data URL
const imageData = `data:image/${imageType};base64,${base64Data}`;

// 4. 存储到数据库
dish.image_data = imageData;
dish.image_type = imageType;
```

## ⚠️ 注意事项

1. **图片大小**: Base64 编码后体积会增加约 33%，建议单张 < 500KB
2. **存储性能**: 图片直接存储在 MongoDB 中，过大会影响查询性能
3. **路径规范**: `image_path` 相对于 `local-deploy/` 目录
4. **容错处理**: 图片缺失不会中断初始化流程

## 🔧 故障排查

| 问题 | 解决方案 |
|------|---------|
| 图片加载失败 | 检查文件路径和权限 |
| 初始化很慢 | 压缩图片文件 |
| Base64 太长 | 减小图片尺寸或质量 |
| 找不到图片 | 使用绝对路径测试 |
