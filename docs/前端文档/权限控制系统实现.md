# 权限控制系统实现文档

## 📋 概述

本系统实现了基于后端角色的完整权限控制机制,包括:
- ✅ 路由级别的权限控制
- ✅ 导航菜单的动态显示/隐藏
- ✅ 无权限访问页面提示
- ✅ 基于后端返回的真实角色数据

## 🏗️ 架构设计

### 1. 后端角色定义 (User.java)

```java
public static final String ROLE_STUDENT = "ROLE_STUDENT";  // 学生
public static final String ROLE_RUNNER = "ROLE_RUNNER";    // 跑腿员
public static final String ROLE_ADMIN = "ROLE_ADMIN";      // 管理员

private List<String> roles = new ArrayList<>(Arrays.asList(ROLE_STUDENT));
```

用户的 `roles` 字段是一个字符串数组,可以包含多个角色。

### 2. 前端权限控制组件

#### ProtectedRoute 组件 (router/ProtectedRoute.jsx)

**功能:**
- 检查用户登录状态
- 检查用户角色权限
- 无权限时显示 `UnauthorizedPage`,而不是重定向

**使用方式:**
```jsx
<ProtectedRoute allowedRoles={[ROLES.STUDENT]}>
  <FoodPage />
</ProtectedRoute>
```

**参数:**
- `children`: 需要保护的页面组件
- `allowedRoles`: 允许访问的角色数组 (可选)

**行为逻辑:**
1. 未登录 → 重定向到 `/login`
2. 已登录但无权限 → 显示无权限页面
3. 已登录且有权限 → 正常显示页面

#### UnauthorizedPage 组件 (pages/UnauthorizedPage.jsx)

**功能:**
- 美观的无权限提示页面
- 显示当前用户角色和所需角色
- 提供返回首页按钮,根据角色跳转到对应页面

**显示信息:**
- 当前用户名
- 当前角色
- 需要的角色
- 友好的错误提示

### 3. 导航菜单控制 (components/Navbar.jsx)

**核心改进:**

```javascript
// 从后端获取用户真实的角色列表
useEffect(() => {
    const storedUser = localStorage.getItem('user');
    if (storedUser) {
        const user = JSON.parse(storedUser);
        const roles = user.roles || [ROLES.STUDENT];
        setUserRoles(roles);
    }
}, [currentUser]);

// 只显示用户实际拥有的角色选项
{userRoles.includes(ROLES.STUDENT) && (
    <button onClick={() => handleRoleSwitch(ROLES.STUDENT)}>
        普通学生
    </button>
)}
```

**行为特点:**
- 角色切换菜单只显示用户**真正拥有**的角色
- 防止用户切换到未授权的角色
- 根据当前激活角色显示对应的导航菜单

### 4. 登录流程 (pages/user/LoginPage.jsx)

**登录时角色设置:**

```javascript
// 从后端响应中获取用户角色
const userRoles = res.data.user?.roles || [];
const defaultRole = userRoles[0] || 'ROLE_STUDENT';
localStorage.setItem("currentRole", defaultRole);

// 根据角色跳转到对应页面
if (defaultRole === 'ROLE_ADMIN') {
    window.location.href = '/admin/runners';
} else if (defaultRole === 'ROLE_RUNNER') {
    window.location.href = '/runner-orders';
} else {
    window.location.href = '/';
}
```

## 🎯 关键实现要点

### 1. 直接URL访问控制

**问题:** 用户直接在地址栏输入无权访问的URL

**解决方案:**
```jsx
// ProtectedRoute 中检查权限
if (allowedRoles && !allowedRoles.includes(currentRole)) {
    return <UnauthorizedPage requiredRoles={allowedRoles} />;
}
```

**效果:** 显示无权限页面,而不是重定向,用户可以清楚知道原因

### 2. 导航菜单隐藏

**问题:** 用户在网页内看到无权访问的菜单项

**解决方案:**
```jsx
// Navbar 中只显示当前角色的菜单
if (currentRole === ROLES.STUDENT) {
    items.push({ key: "/", label: "美食广场" });
}
```

**效果:** 导航栏只显示当前角色可访问的菜单,避免用户困惑

### 3. 角色切换限制

**问题:** 用户尝试切换到未授权的角色

**解决方案:**
```jsx
// Navbar 中只显示用户拥有的角色
{userRoles.includes(ROLES.STUDENT) && (
    <button>普通学生</button>
)}
```

**效果:** 角色切换菜单只显示用户真正拥有的角色

### 4. 状态同步

**问题:** 角色切换后页面状态不更新

**解决方案:**
```jsx
// ProtectedRoute 监听 localStorage 变化
useEffect(() => {
    const handleStorageChange = (e) => {
        if (e.key === 'currentRole') {
            setCurrentRole(localStorage.getItem("currentRole"));
        }
    };
    window.addEventListener('storage', handleStorageChange);
}, []);
```

**效果:** 角色切换后立即更新权限检查
